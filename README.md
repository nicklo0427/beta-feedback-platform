# beta-feedback-platform

## 1. 專案簡介

`beta-feedback-platform` 是一個跨平台 Beta 測試媒合與回饋管理平台。

它的定位不是互刷平台、不是評論交換平台、也不是灌量工具，而是聚焦在「測試者媒合」與「測試流程管理」的產品基礎設施。專案目標是協助開發者在不同平台上，以較一致且可管理的方式完成 Beta 測試招募、任務派發與回饋整理。

目前專案仍處於初始化階段，前後端框架尚未建立，本文件用於說明專案定位、範圍與後續方向。

## 2. 專案目標

本專案希望解決目前 Beta 測試流程中常見的幾個問題：

- 測試者招募來源分散，難以有效找到合適人選
- 不同平台的測試流程零碎，缺乏一致的管理方式
- 回饋格式不一，整理與追蹤成本高
- 缺乏雙向信譽機制，難以建立長期合作品質

本平台的核心目的，是提供一套可標準化的流程，讓開發者能在同一個系統中完成以下工作：

- 招募測試者
- 設定測試條件
- 派發測試任務
- 收集結構化回饋
- 建立雙向信譽

長期而言，本專案希望發展為可擴充到更多平台的跨平台測試基礎設施（testing infrastructure）。

## 3. MVP 範圍

MVP（Minimum Viable Product）階段會聚焦在建立跨平台 Beta 測試流程的最小可行基礎，方向包含：

- 讓開發者發布測試需求並描述基本條件
- 支援測試者媒合與參與流程的基本管理
- 提供測試任務派發與狀態追蹤的基礎能力
- 收集結構化回饋，降低整理與比對成本
- 建立可逐步擴充的雙向信譽資料基礎

MVP 只聚焦在建立平台核心流程，不預設過多進階功能或自動化能力。

## 4. 非目標（Out of Scope）

以下內容目前不屬於本專案現階段目標：

- 互刷、互評、評論交換類型的流量玩法
- 以灌量或操縱指標為目的的功能設計
- 尚未確認需求的進階平台整合
- 與正式商業化營運直接相關的延伸功能細節
- 超出 Beta 測試媒合與回饋管理主軸的工具集合

## 5. 技術棧

目前規劃的技術方向如下：

- Frontend：Nuxt 3 / Vue 3 / TypeScript / Tailwind CSS / Pinia
- Backend：FastAPI / PostgreSQL

此階段僅作為技術方向紀錄，實際初始化與實作細節尚未開始。

## 6. 目前目錄結構

```text
beta-feedback-platform/
├── frontend/
├── backend/
├── tickets/
├── README.md
└── PRD.md
```

各目錄目前用途如下：

- `frontend/`：前端應用程式目錄
- `backend/`：後端服務目錄
- `tickets/`：需求拆解、工作項目與任務追蹤資料夾
- `README.md`：專案首頁文件
- `PRD.md`：產品需求文件（Product Requirements Document）

## 7. 開發原則

本專案現階段採用以下原則推進：

- 先明確定義產品問題與流程，再進入框架與實作初始化
- 優先建立可跨平台複用的核心流程，而不是針對單一平台過度客製
- 以務實、可維護、可擴充為前提，避免過早設計
- 保持需求、文件與實作之間的一致性
- 在功能尚未明確前，不預先增加額外模組與技術複雜度

## 8. 後續規劃

接下來的工作會以需求收斂與結構化整理為主，包含：

- 完成 PRD 與核心流程定義
- 拆解 MVP 所需的功能模組與 ticket
- 明確前後端責任邊界與資料流
- 在需求足夠清楚後，再進行前後端專案初始化

目前專案仍以「定義清楚要解的問題，以及用什麼最小範圍解決它」為優先。
